Separation of Powers: Cognitive vs. Authority

Establishing an Absolute Firewall Between AI Reasoning and Database Execution

Only OS resolves the safety gap by separating the AI system into two distinct planes, ensuring that AI reasoning cannot bypass statutory rules.

01 The Cognitive Plane (LLMs)

Generates text drafts, reasons over citizen queries, and proposes database actions. It has zero system privileges and cannot execute tool calls directly.

02 The Authority Plane (Only OS)

Interprets the proposed parameters and checks them against hard rules written in Only-Lang DSL. It issues short-lived, cryptographically signed capability tokens only if the math balances.

03 The Execution Gateway (Tools)

The databases and APIs (e.g. HelixDB, citizen registries) will only run actions if accompanied by a valid token signature. Any direct, un-gated agent write is physically rejected.
